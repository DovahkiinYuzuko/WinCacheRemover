# 仕様書：WinCacheRemover

## 1. 目的
Windowsシステムおよびすべてのユーザー環境において、削除してもシステムに悪影響を及ぼさないキャッシュや一時ファイルを自動抽出し、安全かつ確実に一括削除することで、ストレージ容量の確保とシステムのパフォーマンス維持を図ります。また、外部設定ファイルにより、プログラム本体を書き換えることなく実行頻度や削除条件を容易にカスタマイズできるようにします。

## 2. 実行環境・前提条件
* **実行環境:** PowerShell 5.1 以上
* **実行権限:** 管理者権限（SYSTEMアカウント、またはAdministrator特権）
* **環境変数の展開:** パス展開において、環境変数はPowerShell標準形式（`$env:SystemRoot`等）を用いて厳格に展開します。
* **構成ファイル:** 本ツールは以下のファイル群で構成され、同一のディレクトリに配置して運用します。
  * `WinCacheRemover.ps1`（実行スクリプト本体）
  * `config.txt`（外部設定ファイル）
  * `last_run.txt`（前回実行日時記録用ファイル：スクリプトが自動生成）

## 3. 外部設定仕様（config.txt）
ユーザーがメモ帳等で容易に設定を変更できるよう、簡易なテキスト形式（`キー=値`）の構成ファイルを採用します。行頭に `#` がある行はコメントとして無視します。

### 設定項目一覧と未定義・破損時のデフォルト値
設定ファイルが存在しない、または記述が破損している場合は、以下のデフォルト値を一律で適用します。

| 設定キー | 説明 | デフォルト値 |
| :--- | :--- | :--- |
| **ExecutionIntervalDays** | クリーンアップを実行する間隔（日数単位） | `7` |
| **LogDirectory** | ログファイルの保存先フォルダパス（空欄時はスクリプトと同一パス） | `""`（スクリプトと同一フォルダ） |
| **MinFileAgeDays** | 削除対象とするファイルの最小経過日数（これ未満の新しいファイルは保護） | `3` |
| **LogRetentionDays** | ログファイルを保持する日数（超過したログは自動削除） | `30` |
| **Delete_[対象名]** | 各削除対象に対する実行フラグ（`true`で削除、`false`でスキップ） | `true`（全対象一律） |

## 4. 削除対象定義および安全性の根拠
複数ユーザー環境を考慮し、固定のユーザーパスではなく、全ユーザーを網羅できるパス設計を採用します。

| 対象名 | 対象パス・パターン | 安全性の根拠・挙動 |
| :--- | :--- | :--- |
| **UserTemp** | `C:\Users\*\AppData\Local\Temp\*` | **安全：** アプリやOSが生成するユーザー単位の一時ファイル。使用後は再利用されません。 |
| **SystemTemp** | `C:\Windows\Temp\*` | **安全：** OSやインストーラーが生成。使用中は自動ロックされるため安全です。 |
| **WindowsUpdateCache** | `C:\Windows\SoftwareDistribution\Download\*` | **安全：** 適用済みのWindows Updateキャッシュ。更新完了後は不要となります。 |
| **InetCache** | `C:\Users\*\AppData\Local\Microsoft\Windows\INetCache\*` | **安全：** IE/レガシーEdgeのキャッシュ。削除してもシステムにより自動再生成されます。 |
| **WebCache** | `C:\Users\*\AppData\Local\Microsoft\Windows\WebCache\*` | **注意：** UWPアプリやEdgeのキャッシュ。システムが掴んでいるDBファイルは自動スキップします。 |
| **CrashDumps** | `C:\Users\*\AppData\Local\CrashDumps\*` | **安全：** アプリクラッシュ時の解析用ログ。開発者によるデバッグ目的以外では不要です。 |
| **UWP_LocalCache** | `C:\Users\*\AppData\Local\Packages\*\LocalCache\*` | **安全：** UWPアプリのキャッシュ領域。アプリが必要に応じて自動再生成します。 |
| **UWP_TempState** | `C:\Users\*\AppData\Local\Packages\*\TempState\*` | **安全：** UWPアプリの一時状態ファイル。削除しても動作に影響ありません。 |
| **Prefetch** | `C:\Windows\Prefetch\*` | **注意：** 起動高速化キャッシュ。削除しても次回起動時に自動再生成されます。 |

## 5. 除外・安全ルール（Exclusions）
* **フォルダ本体の維持 (DoNotDeleteFolders):** 指定されたパスのルートフォルダおよび中間フォルダ自体は削除せず、その配下にあるファイルと子フォルダのみを削除対象とします。
* **ユーザーデータの保護 (NoUserData):** ユーザーが手動で作成したドキュメントやピクチャなどの個人データ領域には一切アクセスしません。

## 6. 機能要件・実装仕様

### 6.1. 同一ディレクトリの構成ファイルのパース
スクリプト起動時、`$PSScriptRoot` を基準として同一フォルダ内の `config.txt` を探索します。ファイルを1行ずつ読み込み、`=` で分割して各設定値を動的に取得します。

### 6.2. 実行頻度の自己判定（last_run.txt 方式）
タスクスケジューラによりスクリプトは毎日起動されますが、実際の削除処理を行うかは専用ファイルを用いて堅牢に判定します。
* スクリプトと同一フォルダ内の `last_run.txt` から前回実行日時を読み込みます。
* 現在時刻との差分が `ExecutionIntervalDays` で指定された日数に満たない場合は、削除処理を行わずに安全にスクリプトを終了します。
* クリーンアップが正常に完了した際、現在時刻のタイムスタンプを `last_run.txt` に上書き保存します。ファイルが存在しない場合は新規作成します。

### 6.3. ファイル経過日数（タイムスタンプ）によるフィルタリング
各削除対象のファイルを走査する際、オブジェクトの最終更新日時（LastWriteTime）または作成日時（CreationTime）を確認します。現在時刻から `MinFileAgeDays` で指定された日数以上が経過している古いファイルのみを削除し、直近に生成された新しいファイルは保護します。

### 6.4. 中間ワイルドカードの事前展開とソート
`Packages\*` や `Users\*` のような中間パスにワイルドカードを含むルートは、`Remove-Item` に直接渡さず、`Get-ChildItem` を用いて事前に対象ディレクトリを厳密に列挙・確定します。
複数ユーザー環境におけるログの再現性と監査性を担保するため、列挙したディレクトリの結果は必ず名前順等でソート（`Sort-Object`）してから削除処理へ渡します。

### 6.5. 実行時エラーの抑制（ロックファイルの扱い）
`WebCache` や `Prefetch` など、システムによって高確率でロックされているファイル群を処理する際、以下の例外が発生してもスクリプトを停止させず、次のファイルの処理へ安全にスキップします。
* `System.IO.IOException`（プロセスが使用中）
* `System.UnauthorizedAccessException`（アクセス権限不足）
* 実装時は `-ErrorAction SilentlyContinue` を適切に配置します。

### 6.6. 軽量ログ出力および固定命名規則によるローテーション
システムの負荷を避けるため、Windowsイベントログへの書き込みは行わず、指定された `LogDirectory` への追記型テキストファイル出力を行います。
* **ログファイル命名規則:** `WinCacheRemover_yyyy-MM-dd.log`（ISO8601形式のタイムスタンプを含める）
* 処理の最後に `LogRetentionDays` を超過した古いログファイルを探索し、自動的に削除（ローテーション）する機能を備えます。

## 7. 自動化要件
* Windowsタスクスケジューラにより、バックグラウンドかつ最上位の特権（SYSTEMまたはAdministrator）で**毎日1回**定期実行するように登録します。
* 実行時の引数：`-NoProfile -ExecutionPolicy Bypass -File "スクリプトパス"`
